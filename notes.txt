///////////
problem is not about seeing the browser. 
The problem is that Jumia's page runs JavaScript that interrogates
 the browser environment before deciding what HTML to serve back.
/////////////
headless=False — browser renders a full visual environment. 
It has screen dimensions, runs JavaScript the way a real user's browser does, 
and passes Jumia's checks because it behaves like a human session.
///////////////
headless=True — browser runs with no graphical environment at all. 
Jumia's JavaScript detects signals like missing screen dimensions, m
issing WebGL, and automation flags baked into the browser — and serves a blocked or stripped page.

error runining jinja2 for the first time:
Jinja2 processes your template BEFORE the browser sees any HTML. 
It does not care that {% %} is inside an HTML comment. 
HTML comments are invisible to browsers — 
but Jinja2 runs first, before the browser is involved at all.
 So Jinja2 tried to execute that empty tag and crashed.

 ** <!-- {% %} is Jinja2 syntax — Flask's templating language --> **
